## 1. Demo 简介

### 是什么

**字帖生成器** 是一款纯前端单页应用（HTML5 + CSS3 + Vanilla JS），双击 HTML 文件即可在浏览器中运行。无需服务器、无需安装、无需联网。输入汉字后自动生成带拼音标注、组词提示、笔画分解的描红字帖，支持通过浏览器原生打印输出矢量 PDF。

### 面向谁

- **中小学语文教师** — 快速生成课堂练习字帖，自定义练习内容
- **家长** — 为孩子定制课后练字纸，随时打印
- **书法爱好者** — 自由切换 9 款书法字体，个性化练习
- **任何想写好汉字的人** — 正如作者所说："自从用了电脑，打字技术增长，但字却越来越丑"

### 核心功能

| 功能 | 说明 |
|------|------|
| 智能描红字帖生成 | 输入汉字自动生成田字格描红字帖，支持拼音标注（含声调） |
| 组词辅助 | 内置 4000+ 词库，自动为每个汉字提供组词参考 |
| 笔画分解 | 基于 Hanzi Writer 输出 SVG 笔画分解，辅助正确书写 |
| 矢量 PDF 输出 | 两种方式：浏览器打印另存为 PDF + Puppeteer 一键矢量 PDF |
| 9 款书法字体 | 内置姜浩硬笔楷书、华文楷体、方正仿宋等，支持上传自定义字体 |
| 极光毛玻璃 UI | 日间/夜间双主题，打印时自动隐藏面板 |

## 2. Demo 创作思路

### 灵感来源

作为一名书法练习者，我在日常练字中遇到几个痛点：市面上的字帖内容固定、无法定制；在线字帖工具依赖网络且常收费；打印出来的字帖往往是位图、模糊不清。于是想到用前端技术，做一个**完全离线、自由定制、高清矢量输出**的字帖工具。

### 想解决的问题

1. **内容不可定制** — 传统字帖内容是固定的，无法针对性练习某个课文或诗词
2. **依赖网络** — 在线工具需要联网，教室或家中网络不稳定时无法使用
3. **输出质量差** — 很多工具截图生成位图，放大后模糊，不适合打印
4. **缺少教学辅助** — 仅有字形，没有拼音、组词、笔画分解等教学信息

### 为什么做这个方向

- **真实需求**：汉字书写是中华文化的基础教育，教师和家长每天都需要练习纸
- **技术可行**：现代浏览器已支持 FontFace API、CSS Grid、window.print() 等关键能力
- **差异化**：市面上缺少"离线 + 矢量 + 教学辅助"三位一体的免费工具
- **个人积累**：练字过程中积累了多款字体的使用经验，了解用户真实需求

> 我自从用了电脑，打字技术增长，但字却越来越丑；自从练习写字之后，用小楷毛笔在 A4 复印纸上都能写出这个水平的字了：

## 4. TRAE 实践过程

本项目全程使用 **TRAE IDE** 完成核心逻辑开发、调试与迭代优化。以下是开发关键步骤。

### 技术架构

| 技术点 | 方案 | 说明 |
|--------|------|------|
| 架构 | 纯 HTML5 + CSS3 + Vanilla JS | 单页面，零依赖运行 |
| 拼音转换 | `pinyin-pro`（MIT） | 自动标注声调 |
| 组词查询 | `cnchar` + 4000+ 词库（MIT） | 智能组词推荐 |
| 笔画渲染 | `Hanzi Writer`（MIT） | SVG 笔画分解 |
| 字体加载 | `FontFace API` 动态注册 | 支持用户上传字体 |
| PDF 导出 | `window.print()` + Puppeteer | 双轨方案 |
| 主题切换 | CSS 变量 + `data-theme` | 日间/夜间 |
| 开发工具 | **TRAE IDE** | AI 辅助编码、调试 |

### 关键开发步骤

**步骤一：需求分析与技术选型**

通过 TRAE 的多智能体并行任务规划能力，将字帖生成器拆解为：拼音转换模块、组词查询模块、田字格渲染模块、笔画分解模块、PDF 导出模块。TRAE 帮助分析各模块的技术方案和依赖关系。

**步骤二：核心功能实现**

- 使用 TRAE 的 AI 辅助编码功能，快速实现拼音转换（pinyin-pro 集成）和组词查询（cnchar 集成）
- 田字格使用 CSS Grid 布局动态生成，每个格子包含米字格线、描红字、拼音标注
- 笔画分解利用 Hanzi Writer 输出 SVG，嵌入到每个字格中

**步骤三：UI 设计与主题系统**

- 极光毛玻璃风格 UI，采用 CSS 变量驱动的日间/夜间双主题
- TRAE 辅助完成复杂的 CSS 动画和渐变效果
- 打印时自动隐藏控制面板，确保输出整洁

**步骤四：PDF 矢量输出双轨方案**

- 方案一：浏览器 `window.print()` 另存为 PDF，全平台通用
- 方案二：Puppeteer 无头浏览器生成矢量 PDF，文字可选可复制
- TRAE 辅助解决了字体加载时序、keep-alive 请求合并、跨平台编码等关键问题

**步骤五：离线打包与发布**

- 将所有 JS 库（pinyin-pro、cnchar、Hanzi Writer）内嵌到 HTML
- 拼音字体 base64 内嵌，汉字字体使用 FontFace API 加载
- 编写 Windows/Linux 一键启动脚本，实现"傻瓜化"操作

### 开发踩坑与经验

1. **字体加载时序**：Puppeteer 生成 PDF 时，`domcontentloaded` 事件在字体加载前触发，导致 PDF 乱码。解决方案：改用 `networkidle0` + 显式 `document.fonts.check()` 验证。

2. **HTTP keep-alive 请求合并**：浏览器在 keep-alive 连接上重复发送请求，导致多个 JSON 请求体被合并，`JSON.parse()` 失败。解决方案：三层防御（`Connection: close` + `safeJsonParse()` + 请求去重）。

3. **Windows 批处理编码**：`.bat` 文件中文注释在 cmd.exe 的 GBK 编码下被错误解析，导致命令截断。解决方案：添加 `chcp 65001` 切换到 UTF-8 代码页。

4. **PowerShell 安全警告**：每次运行弹出安全提示。解决方案：预设 `PSDefaultParameterValues` 全局参数。

## 5. 文件结构

```
distribution/
├── 字帖生成器.html          # 主程序（独立 HTML，JS+拼音字体内嵌）
├── 启动Puppeteer.bat         # Windows 一键启动脚本
├── 启动Puppeteer.ps1         # PowerShell 启动脚本
├── 启动Puppeteer.sh          # Linux/macOS 启动脚本
├── puppeteer-pdf.js          # Puppeteer PDF 矢量生成脚本
├── package.json              # Node.js 依赖配置
├── README.md                 # 完整使用文档
├── CHANGELOG.md              # 更新日志
├── 参赛文档.md               # 本文档

    ├── 姜浩硬笔楷书.ttf       # 默认汉字字体
```
